Subversion Bundle README

What you currently can do with this Bundle

Log
active file
Yes
shows you the commit messages for the current file. configurable via some shell variables, see below.
Blame
active file
Yes
shows you who wrote which line in what revision. you can click a line to jump to this line in TM.
Status
directory of active file
No
gives you an overview over what files changed.
Info
selected files in drawer
No
gives you a list of information about this files. (url, last changed, etc)
Diff With Newest
active file
Yes
shows you a colored diff against the newest available revision of this file in the repository. (HEAD)
Diff With Working Copy
active file
No
shows you a colored diff against the working copy revision, also called the pristine copy. (BASE) it basicly shows you all your changes.
Diff With Previous Revision
active file
Yes
shows you a colored diff against the previous revision of this file. (PREV)
Commit
active file
Yes
pop up a fancy message window to describe your changes and then send your changes to the repository.
Add to Repository
active file
No
adds the selected files to the repository.
Update to Latest
active file
Yes
updates the currently selected files to the newest available.
Update Entire Project
project
Yes
updates everything.

Configuration Options

$TM_SVN
svn
the path to your svn executable.
$TM_SVN_LOG_RANGE
BASE:1
the default range to query for log messages.
$TM_SVN_LOG_LIMIT
7
the number of messages, to actually show.
