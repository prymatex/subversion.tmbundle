Subversion Bundle

Commands

Log
selected project files/folders or the active file if it doesn't belong to a project
Yes
Displays the commit message history for the selected files. Some parameters are configurable via shell variables; see the Configuration Options section below.
Blame
active file
Yes
Displays a line-by-line history of the file, showing you who wrote which line in what revision. Click a line to jump to it in an editor window.
Status
directory of active file
No
Displays a list of files with changes in your working copy, along with the type of change for each file.
Info
selected project files/folders or the active file if it doesn't belong to a project
No
Displays detailed information about the selected files, including the type of file, who last changed the file, the date file was last changed, the repository URL to the file, and other information. Some parameters are configurable via shell variables; see the Configuration Options section below.
Diff With Newest (HEAD)
active file
Yes
Displays the differences between the active file and the newest available revision of the file on the server. Equivalent to svn diff -rHEAD.
Diff With Working Copy (BASE)
active file
No
Displays the differences between the active file and an unaltered, pristine copy of the file at the same revision. Equivalent to svn diff -rBASE.
Diff With Previous Revision (PREV)
active file
Yes
Displays the differences between the active file and the previous revision of the file. Equivalent to svn diff -rPREV.
Commit
selected project files/folders or the active file if it doesn't belong to a project
Yes
Commits your changed files to the repository. A custom window appears, in which you enter the description of your changes and you may choose to exclude files from the commit. If no files are selected or active, this command does nothing. If the target files have no local changes, nothing happens.
Add to Repository
selected project files/folders or the active file if it doesn't belong to a project
No
Adds the targets to the repository.
Update to Latest
selected project files/folders or the active file if it doesn't belong to a project
Yes
Updates the targets with the newest changes from the repository.
Update Entire Project
project
Yes
If there is a project open, updates all of the files and folders in the project with the newest changes from the repository.

Configuration Options

$TM_SVN
svn
the path to your svn executable.
$TM_SVN_LOG_RANGE
BASE:1
the default range to query for log messages.
$TM_SVN_LOG_LIMIT
9
the number of messages, to actually show.
$TM_SVN_INFO_HIDE
nil (nothing)
here you can adjust what keys of the info stream you don't want to see, it is case insensitive and a comma (,) seperated list. if you enter a *, it is assumend that you want to hide all vars, if so, you can make some values visible again with $TM_SVN_INFO_SHOW.
$TM_SVN_INFO_SHOW
nil (everything else)
this is also a commma seperated list of keys, it just makes sense if you did hide all with the above var. * has no special meaning here. everything you enter here will be shown if you want to hide everything else.
